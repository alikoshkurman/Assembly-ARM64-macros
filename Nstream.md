# Набор сетевых макросов для AArch64 Linux

Коллекция высокопроизводительных макросов и констант для GNU Assembler (GAS), реализующая низкоуровневую работу с сокетами на архитектуре **ARM64 (AArch64) Linux** напрямую через системные вызовы (`svc #0`).

---

## Важное правило передачи аргументов

В этом наборе макросов аргументы делятся на два типа. Не путай их при вызове:

1. **`name` / `name1` / `client_name`** — это **текстовые имена переменных в памяти** (метки). Сюда передаются придуманные тобой названия сокетов (например, `my_server`, `client_socket`). Макрос сам создаст их в памяти через `save.reg`.
2. **`reg` / `reg_addr` / `reg_len`** — это **физические регистры процессора** (например, `x11`, `x21`, `x22`). Они используются только там, где нужны немедленные вычисления или возвращаются динамические адреса.
3. **`txt`** — текстовая строка в кавычках (например, `"Hello"`).
4. **`port` / `ip` / `d`** — предкомпилированные константы (например, `web.srvs`, `any`, `IPv4`).

## Справочник по передаче аргументов в макросы

### 1. Управление сокетами

* **`net.open.fd reg, o, t, r`**  
  * *Что передавать:* В `reg` — **рабочий регистр** (например, `x11`). Остальное — константы.  
  * *Пример:* `net.open.fd x11, IPv4, SOCK.S, ITCP`

* **`net.bind name, name1, d, port, ip`**  
  * *Что передавать:* В `name` — имя сокета в памяти, в `name1` — имя переменной для результата. **Регистры совать нельзя.**  
  * *Пример:* `net.bind my_server, bind_res, IPv4, web.srvs, any`

* **`net.listen name, blog, name1`**  
  * *Что передавать:* В `name` — имя сокета в памяти, в `name1` — имя переменной для результата. В `blog` — размер очереди со знаком `#`.  
  * *Пример:* `net.listen my_server, #5, listen_res`

* **`net.open name, port, ip, ...`**  
  * *What к чему:* В `name` — придуманное **имя твоего сервера**. Макрос сам создаст переменную в памяти, создаст сокет, привяжет его и запустит прослушивание.  
  * *Пример:* `net.open my_web_server, web.srvs, any`

* **`net.close name`**  
  * *Что передавать:* Только **имя сокета в памяти**, который нужно закрыть.  
  * *Пример:* `net.close my_web_server`

---

### 2. Соединения и передача данных

* **`net.connect name, name1, d, port, ip`**  
  * *Что передавать:* В `name` — имя твоего сокета, в `name1` — имя переменной под статус соединения.  
  * *Пример:* `net.connect my_socket, conn_status, IPv4, HTTP, localhost`

* **`net.acpt name, client_name, need`**  
  * *Что передавать:* В `name` — имя слушающего сервера из памяти. В `client_name` — **новое имя** для сокета подключившегося клиента. В `need` — флаг `yes` или `no`.  
  * *Пример:* `net.acpt my_web_server, web_client, yes`

* **`net.send.to name, name1, txt, d, port, ip`**  
  * *Что передавать:* В `name` — имя UDP сокета, в `name1` — имя переменной для результата, в `txt` — строку в кавычках.  
  * *Пример:* `net.send.to my_udp, send_res, "Ping", IPv4, DNS, router`

* **`net.write txt, name`**  
  * *Что передавать:* В `txt` — **текст в кавычках**, в `name` — **имя сокета в памяти**, куда слать данные.  
  * *Пример:* `net.write "страница", web_client`

---

### 3. Чтение и веб-абстракции

* **`net.sawc txt, name, head`**  
  * *Что передавать:* В `txt` — HTML код в кавычках, в `name` — имя твоего готового сервера из памяти.  
  * *Пример:* `net.sawc "<h1>Works!</h1>", my_web_server`

* **`net.read name, reg_addr, reg_len`**  
  * *Исключение:* Этот макрос принимает и имя, и регистры, так как буфер выделяется динамически.  
  * *Что передавать:* В `name` — имя сокета, откуда читаем. В `reg_addr` — **регистр процессора** для адреса буфера, в `reg_len` — **регистр процессора** для длины данных.  
  * *Пример:* `net.read web_client, x21, x22` (после вызова в `x21` лежит адрес данных в куче, в `x22` — длина).

## Пример полной программы (`main.s`)

```assembly
.include "network_macros.s"

.global _start
.text
_start:
    // Передаем аргументы: имя переменной памяти, порт, IP
    net.open my_server, web.srvs, any

    // Передаем аргументы: строку текста и имя сервера из памяти
    net.sawc "<h1>Hello from ARM64 Assembly!</h1>", my_server

    // Выход из программы
    mov x0, #0
    mov x8, #93
    svc #0
```
