# Полная документация макросов AArch64 (ARM64) для Linux

Этот набор макросов абстрагирует системные вызовы Linux для выполнения базовых файловых операций, скрывая сложности работы со стеком, регистрами и секциями памяти.

---

## 1. Константы и права доступа (Флаги)

Команды `.set` определяют числовые константы, которые комбинируются с помощью побитового сложения (`|`).

### Режимы открытия файлов (Flags)
* `read`: `0` (`O_RDONLY`) — только для чтения [man7.org].
* `write`: `1 | 64` (`O_WRONLY | O_CREAT`) — для записи с созданием, если файла нет [man7.org].
* `creat`: `64` (`O_CREAT`) — создать файл при отсутствии [man7.org].
* `trunc`: `512` (`O_TRUNC`) — очистить файл при открытии [man7.org].
* `append`: `1025` (`O_WRONLY | O_APPEND`) — дозапись в конец файла [man7.org].

### Таблица прав доступа (POSIX Permissions)
Используются при создании нового файла через побитовый сдвиг (`<<`).

| Группа пользователей | Чтение (`r`) | Запись (`w`) | Выполнение (`x`) |
| :--- | :--- | :--- | :--- |
| **Владелец (User)** | `ar = (4 << 6)` | `aw = (2 << 6)` | `ax = (1 << 6)` |
| **Группа (Group)** | `gr = (4 << 3)` | `gw = (2 << 3)` | `gx = (1 << 3)` |
| **Остальные (Other)** | `otr = 4` | `otw = 2` | `otx = 1` |

> **Пример:** Чтобы задать права `rwxr-x---` (в восьмеричной системе `0750`), передайте макросу комбинацию:
> `(ar | aw | ax | gr | gx)`

## 2. Справочник по макросам

### `file.open reg, path, flags, mode, desc`
Открывает или создает файл. Безопасно сохраняет используемые регистры в стек (`sp`) и восстанавливает их.
* **`reg`**: Регистр, куда запишется дескриптор файла (например, `x19`).
* **`path`**: Строка-путь к файлу в кавычках.
* **`flags`**: Флаг открытия (`read`, `write`, `append`).
* **`mode`**: Комбинация прав доступа (например, `#0760`).
* **`desc`**: Базовый дескриптор директории. По умолчанию равен `#-100` (`AT_FDCWD`), что означает текущую папку [man7.org].

### `file.close reg`
Закрывает ранее открытый файл.
* **`reg`**: Регистр, содержащий файловый дескриптор.

### `start name` 
Открывает точку входа в файле.

### `file.write txt, reg`
Записывает текстовую строку в файл. Длина строки вычисляется автоматически.
* **`txt`**: Текст в кавычках. Символ `"` внутри строки экранируется через косую черту и код `\042`.
* **`reg`**: Регистр с файловым дескриптором.

### `file.read file, name, size, len`
Выделяет в секции неинициализированных данных (`.bss`) буфер с именем `name` размером `size` байт и считывает туда данные из файла.
* **`fd`**: Регистр с дескриптором открытого файла.
* **`name`**: Имя создаваемого буфера (будет доступно во всей программе).
* **`size`**: Количество байт для чтения.
* **`len`**: Регистр, куда запишется количество фактически считанных байт. **Нельзя** передавать `x0`, `x1`, `x2` и `x8`.

### `printb name`
Выводит на экран name, пример:
```
file.read file, name, сколько_прочитать, reg2
printb name
```

### `file.remove path, fd`
Удаляет файл с диска по указанному пути.

### `trunc.file file`
Быстро очищает содержимое существующего файла до 0 байт.

### `save.fd reg, name` / `load.fd reg, name`
Сохраняет файловый дескриптор из регистра в ячейку памяти `name` (8 байт в `.bss`) и считывает его обратно для освобождения регистров.

## 3. Критические ограничения и запреты

Нарушение этих правил приведет либо к ошибке компиляции, либо к аварийному завершению программы (`Segmentation fault`).

* **Запрещено передавать `x0`, `x1`, `x2`, `x8` в параметр `len` (`file.read`)**
  Эти регистры затираются системным вызовом. Компилятор выдаст синтаксическую ошибку сборки. Используйте вместо них `x3`–`x7` или `x19`–`x28`.
* **Запрещено забывать закрывать файлы (`file.close`)**
  У операционной системы Linux есть лимит на количество открытых файлов. Если лимиты исчерпаются, новые вызовы `file.open` начнут возвращать ошибку.
* **Запрещено использовать двойные кавычки `"` внутри текста `file.write` напрямую**
  Ассемблер запутается в границах аргумента. Экранируйте кавычки через код: `\042`. Например: `file.write "Say \042Hello\042", x19`.
* **Запрещено использовать макрос `file.write` для чтения данных**
  Текст жестко зашивается в секцию `.rodata` (память только для чтения). Попытка записи поверх этой памяти вызовет крэш программы.
* **Запрещено вызывать макросы в секциях данных (`.data`, `.bss`, `.rodata`)**
  Макросы разворачиваются в исполняемый код (`mov`, `svc`, `str`). Вызывайте их строго внутри секции кода `.text`.
* **Запрещено повторно использовать одно имя буфера в `file.read` с разным размером**
  Макрос резервирует место под конкретным именем. Повтор вызовет ошибку компилятора `symbol уже определен`. Дарите буферам уникальные имена.

## 4. Практический пример использования

```armasm
start main
    // 1. Создаем файл "data.txt" с флагом write и правами rwxrwx---
    file.open x19, "data.txt", write, (ar|aw|ax|gr|gw|gx)
    
    // 2. Записываем текст в файл
    file.write "Initial line\n", x19
    file.close x19

    // 3. Очищаем файл (размер станет 0 байт)
    trunc.file "data.txt"

    // 4. Открываем для записи в конец (append)
    file.open x19, "data.txt", append
    file.write "New text after truncation\n", x19
    file.close x19

    // 5. Открываем для чтения (read)
    file.open x19, "data.txt", read

    // 6. Читаем до 64 байт в буфер 'my_buffer'. 
    // Число считанных байт запишется в x22
    file.read x19, my_buffer, 64, x22
    file.close x19

    // 7. Печатаем уведомление на экран и выходим с кодом 0
    print "File processed successfully!"
    exit #0
```
